A

Application

The user applies to join a course run by a training provider. This happens when they register with the Department for Education (DfE).

Application ID

Each application has a unique ID. You can use this ID to find a specific application. Application ID example: 0594fe7b-fcd2-4f3f-9e8d-17ab5538a2b9.

Application programming interface (API)

A set of tools and protocols that allow different software systems to securely connect with ours to:

  • collect new applications
  • accept or reject applications
  • update application, participant and declaration details
  • submit declarations and outcomes

Application status / training status

The status shows if a user is currently taking part in an NPQ course. The status can be:

  • 'Active' – they're currently doing the course
  • 'Deferred' – they've taken a break from the course (only possible if a start declaration has been submitted)
  • 'Withdrawn' – they've stopped training and are no longer doing the course

C

Clawbacks

The total amount of money reclaimed after paid declarations were voided.

Cohorts

These are groups of users who start an NPQ course and progress through it at the same time. They usually run on an annual basis and are made up of different training schedules.

Continuous professional development (CPD)

Continuing Professional Development (CPD) is a programme run by the Department for Education (DfE). It helps teachers and school leaders access high-quality training.

Course ID

Each course has a unique identifier known as a course ID.

Course identifier

This is the name of the NPQ course the application is for, written in the format used by the API. For example: npq-headship. Use this format when searching for or retrieving course data through the API.

Course name

The name of the course linked to the application. For example: NPQ for Headship (NPQH).

Course provider (CP)

DfE-accredited organisations that deliver teacher and school leader training, such as NPQs.

'Created at' (applications tab)

This field provides the date when an application was first created. For example: 1 Jul 2025 at 3:11pm.

'Created at' (declarations)

This field provides the date when a declaration was first logged in our system. For example: 1 Jul 2025 at 3:11pm.

D

Declaration ID

The unique ID for a declaration.This ID can be used to search for a specific declaration through the API, or to find an application.

Declaration date

The date a milestone was reached.

Declaration type

The type of declaration made for the course.It shows whether the training has started, is in progress, or has been completed.

Declarations

A marker that shows a milestone has been reached in an NPQ training programme.

Delivery partner (DP)

An organisation that supports a provider by delivering parts of a training or induction programme, such as the Early Career Framework (ECF) or NPQs.

E

Early headship coaching offer (EHCO)

Program that provides free, one-to-one coaching and peer support for new headteachers within their first five years of headship to help them effectively apply leadership knowledge in their specific school context.

Early years (EY)

The phase of education covering children from birth to age 5, including nursery, preschool, and reception.

Edit details (cohorts)

Console users can edit cohort details including the start year, registration date and the funding cap.

Eligible for funding

The user meets the funding criteria and is eligible for scholarship funding to cover their course fees.

Back to top

F

Funded place

A funded place confirms that the applicant meets the provider’s course requirements, is eligible for funding, and has been offered a funded place on a course by the provider.

Funding cap

If this is turned on in the admin console for a cohort, then providers must confirm which applications will be assigned DfE funding at the point of acceptance.

Funding choice

If an application is not eligible for funding, the applicant chooses how they want to pay. Options include:

  • self-funded
  • funded by their school
  • shared with their workplace
  • other

H

Headteacher status

This field is to indicate whether the user is a headteacher or not at the point of registration, or if the user will be granted headteacher status during their NPQ training.

I

Initial teacher training (ITT)

This is the training a person does to become a qualified teacher.

Initial teacher training (ITT) lead mentor

If a user is an ITT lead mentor, they may be eligible for funded NPQ courses focused on leading teacher development (NPQ LTD).

Initial teacher training (ITT) provider

Organisations that are accredited by DfE to deliver ITT training.

In review

Some NPQ registrations need to be manually reviewed by funding policy staff. This happens when a decision is needed on whether the applicant qualifies for scholarship funding.

M

Manual adjustments

Manual adjustments made by an admin user to financial statements. These changes affect the total amount shown in the statement.

Multi-academy trust (MAT)

A group of academies governed by a single legal entity that oversees finances, standards, and school improvement across its member schools.

N

National professional qualifications (NPQs)

Accredited leadership development programmes for teachers and school leaders at various stages of their careers.

O

Output payment

The total amount paid for all milestones reached. This contributes to the overall fee shown in the financial statement, along with the service fee.

Output payment date

The date that the payment for a statement is due.

Output payment deadline

The deadline for submitting a declaration so it can be included in a statement.

P

Private childcare provider URN

The unique reference number (URN) for a private childcare provider, as registered with Ofsted. Also known as an 'OFSTED URN'.

Provider approval status

Shows whether a provider has accepted or rejected a user’s application to join their course.

Paid and unpaid status (financial statements)

Shows whether payment has already been made for a statement, or if it’s still pending.

Q

Qualified teacher status (QTS)

A legal certification required to teach in most state schools in England, awarded after completing initial teacher training.

Back to top

R

Registration start date

This is the date that the Register for an NPQ service opens to accept registrations for a cohort.

Remove (cohorts)

Admin users can remove cohorts. Once removed, the admin user cannot restore the cohort.

Back to top

S

Schedule identifier

Identifies which schedule the user is on for their NPQ course. Within a cohort year, there can be multiple schedules — for example, autumn or spring. Schedules may also differ by course type, such as specialist or leadership.

School UK Provider Reference Number (UKPRN)

Given to higher education organisations to help identify the institution.

School unique reference number (URN)

This a six-digit number used by the UK government to identify educational organisations in the UK.

Service fees (statements)

Contributes to the total figure of a statement, along with the total output payment - this represents the amount paid each month over an agreed installment.

Special educational needs and disabilities co-ordinator (SENDCo)

A teacher who oversees and coordinates support for pupils with special educational needs or disabilities (SEND).

Special educational needs co-ordinator (SENCo)

Another term for SENDCo; used interchangeably to describe the same role in supporting pupils with additional needs.

Start year (cohorts)

The year that a cohort begins.

State (declarations)

This refers to the current stage or condition of a payment linked to a submitted declaration. Possible statuses include:

  • Submitted – declaration has been received.
  • Eligible – declaration meets the criteria for payment.
  • Ineligible – declaration does not meet the criteria for payment.
  • Voided – declaration has been cancelled or invalidated.
  • Payable – payment is approved and ready to be issued.
  • Paid – payment has been successfully made.
  • Awaiting clawback – A repayment is expected due to overpayment or error.
  • Clawed back – payment has been recovered.

Statements

We create and send financial statements to course providers. These statements explain what we owe to each provider.

Statement ID

Each statement has a unique ID. You can use this ID to find and track specific statements.

T

Teacher reference number (TRN)

A unique, 7 digit identification number that helps identify users in the education system in England. This is needed to register for an NPQ and to download certificates.

Teacher reference number (TRN) not verified

The user's teacher reference number (TRN) has not been verified in the Teaching Record System (TRS).

Teacher reference number (TRN) verified

The user's teacher reference number (TRN) has been verified in the Teaching Record System (TRS).

Teaching regulation agency (TRA)

This is a government organisation responsible for:

  • handling teacher misconduct cases
  • awarding qualified teacher status (QTS)
  • keeping the official record of qualified teachers in England

Teaching record system (TRS)

A government-maintained record of teachers in England who have been awarded qualified teacher status (QTS).

Teaching school hub (TSH)

A regional centre designated by the DfE to lead on teacher training, professional development, and school improvement in a local area.

U

Unique reference number (URN)

A unique identifier assigned to each educational establishment in England, such as schools and academies, for use in government systems.

'Updated at' (applications)

The date in which the latest change or update was made to an application (for example, an acceptance of an application).

'Updated at' (declarations)

The date in which the latest change or update was made to a declaration (for example, voiding a declaration).

Upload contracts

The following information needs to be uploaded to the admin console to create contracts: lead provder names, course identifiers, recruitment targets, fee per participant, service fee installments, monthly fees.

Upload statements

To upload a statement a CSV file with the year, month, deadline date, payment date and output fee information must be uploaded.

User ID

Each user has a unique ID. This helps you search for and identify individual users. A user might have several applications—or none at all.
